# Tasks Proposition: agentcore / enthusiast Architecture Split

> **Note:** This is a **proposed** task breakdown generated by **Claude (Opus 4.8)** based on
> [`ai-agent-core-extraction.md`](./ai-agent-core-extraction.md). It is a starting point for
> planning, not a committed plan — epics and tasks should be reviewed, re-scoped, and re-estimated
> before being turned into tickets. Tasks are grouped into epics at a "basic" granularity
> (no hour-level subtasks).

---

## Epic 0 — Foundation (critical path, blocks everything)

| # | Task | Spec reference |
|---|---|---|
| 0.1 | Bootstrap **agentcore** repo — runnable Django project, `settings_override.py` mechanism | Two Repositories |
| 0.2 | **Extract `agentcore-common`** — pure ABCs: `BaseAgent`, `BaseAgentBuilder`, `BaseTool`, `BaseInjector`, `BaseVectorStoreRetriever`, `BaseIntegration`, `BaseEmbeddingChunk`, `DomainEntity`, `EntityField` | Package Structure, dependency graph |
| 0.3 | Plugin-loading mechanism — read `AGENTCORE_*_PLUGINS` lists + registries (agent, builder, integration, entity, agentic-execution) | Plugin Registration |

> 0.2 is the linchpin — hardest task, since it requires cleanly separating pure framework
> interfaces from domain knowledge. Everything else builds on it.
